將Hugo網站放置在github page上

想要擁有屬於自己的website嗎？

Posted by chinchio on Saturday, July 16, 2022

緣由

用hugo建一個blog，把自己的學到的東西都寫上去，把他當作是學習日記好了

Install

Git

這個不用多說吧，直接在Git官網下載然後安裝即可

Hugo for Windows

進入這個hugo release網站 我個人是偏好是下載binary檔案然後放在env path裏面，如果不喜歡也可以到hugo installing 跟隨官方步驟使用Chocolatey來以command line的方式安裝hugo

下載hugo binary

因為我是要在Windows x64上面使用 所以在這裏我會下載hugo_0.101.0_Windows-64bit.zip 其他人按照自己的系統版本選擇吧！

解壓縮並設定path

打開hugo_0.101.0_Windows-64bit.zip後 可以看到裏面有三個檔案 我們需要的就只有hugo.exe 我個人習慣是把這些binary file（如ffmpeg)等都放在C:\Program Files\<program_name>裏面 打開Environment Variable的設定工具 把hugo的path加到path裏面去

測試是否正常

然後打開terminal，直接輸入hugo version看看有正確輸出hugo version相關msg 如果沒有那可能你設定path時出錯了

Github page

Github page引言

Host on GitHub | Hugo 要把網頁架設在Github上就會用到GitHub Pages(廢話)

之前我也曾經在其他Github account用hugo作為blog 那時只需要把生成的html && image && css %% js push到main branch即可

但如果跟隨hugo的tutorial是利用Github Actions讓Github自動把你寫好的*.md生成web page並把該生成的檔案放在gh-pages這條branch上

所以才會用到gh-pages的branch (我猜，如果有寫錯的話可以在留言區提醒 如果我有弄留言區的話 XD)

因為時間關係（誤），我已經把以username建立的repo建立好，那現在先把該repo clone 到 local再說吧

如果不知道為甚麼要把repo命名為xxx.github.io，可以參考Github|Github Pages裏面有說明

hugo newsite

當然你要先用hugo建立一個網站才可以讓github幫你托管網頁，所以跟隨官方tutorial Windows run command hugo new site

hugo new site <newsite>

然後再去hugo themes下載你喜歡的主題，安裝主題及裏面的exampleSite不懂請參考其他的教學

但要注意如果你是直接把hugo themes的theme git clone下來， 有機會會在github actions的步驟裏有 fatal: No url found for submodule path 'themes/hugo-theme-cleanwhite' in .gitmodules 的error，所以要把theme裏面的.git刪除掉

如果你安裝主題沒有出任何錯的話應該可以在 <newsite> folder 下指令

hugo server

應該就可以看到像這樣類似的畫面

最後把<newsite>的檔案放在剛才clone下來的repo的folder裏面(像下圖)

使用Github Actions

做完以上的步驟，接下來學官方的教學新增一個 gh-pages.yml 來讓github生成webpage到gh-pages branch裏面 Build Hugo With GitHub Action 好了 完成上述步驟 就來上傳github的三步曲add, commit還有 push 你push完後應該可以在github actions有以下的畫面 (All green is good!)

來一個結尾

好了 經過這麼多工序後，終於來到最後一步

就是要把source的選項改成gh-pages的branch,要不然你打開<username>.github.io只有你main branch那個空空如也的readme.md

好了 大家快跟我一起來探討人生的意義吧
